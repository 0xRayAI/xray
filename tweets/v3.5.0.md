## Release note (docs / GitHub)

### Headline

**3.5.0 ships two layers.** Honesty + SSOT (3.4.x, all bridges) plus **loop enforcement runtime on Grok Build only**. Don't say "the suit" without naming the host.

### Two layers in one tarball

| Layer | What | Hosts |
|-------|------|-------|
| **Orchestrator brain** | Honest deferrals, `delegations[]`, `pending-delegations.json` SSOT, lead-dev plan persistence, memory routing | All 4 bridges |
| **Governance OS runtime** | `checkPendingDelegationGate`, `checkSubagentGate`, PostToolUse clear | **Grok only** |

### What's live (Grok Build)

| Capability | What it does |
|------------|--------------|
| **Auto-chain bridge (PR1)** | PreToolUse deny on writes while pending; PostToolUse logs + Task clears `pending-delegations.json` |
| **Spawn todo persistence (PR2)** | Plan enforced while todos open; spawn must target next plan todo |
| **Grok hooks** | PreToolUse + PostToolUse via `npx 0xray grok install --force` |

**Operating model (enforced on Grok):**

```
intake → analyze-complexity → orchestrate (defer) → [gate] → Task → test → approve
```

### What's live (all bridges)

| Capability | What it does |
|------------|--------------|
| **`delegations[]`** | Orchestrate surfaces what was deferred |
| **SSOT state** | `pending-delegations.json` written on defer (read for blocking: Grok only today) |
| **8/8 behavior verify** | `verify-orchestrator-behavior.mjs` — any consumer with `0xray@3.5.0` |

### Per-host "suit worn"

| Host | Honest orchestrator | Loop enforcement | Verify bar |
|------|---------------------|------------------|------------|
| Grok | ✅ | ✅ Pre/PostToolUse | 32/32 |
| Hermes | ✅ | ⚠️ codex/quality, not delegation | ~10 bridge checks |
| OpenCode | ✅ | ⚠️ codex plugin, not delegation | ~9 |
| OpenClaw | ✅ | ⚠️ observability only | ~9 |

### Install (consumer)

```bash
npm install 0xray@3.5.0
npx 0xray grok install --force
```

Restart Grok MCP sessions after install — hooks do not hot-reload.

**With Repertoire memory routing:**

```bash
npm install @0xray/repertoire@0.1.3
npx repertoire-verify-suit   # 32-check harness, npm-only (0.1.3 hoist fix)
```

Enable in `.xray/features.json`:

```json
"multi_agent_orchestration": {
  "lead_dev_mode": true,
  "auto_chain_delegations": true
}
```

Opt out: `"auto_chain_delegations": false` disables PR1 + PR2 enforcement (Grok hooks).

### 3.5.1+ roadmap

**Multi-host gate parity** — extract `delegation-gate.ts` from `grok-hook-utils.js`; wire Hermes `pre_tool_call`, OpenCode `tool.execute.before`, OpenClaw tool-before; per-host 4/4 fixtures.

**Can parallel:**
- **#3 Session trail** — all hosts (MCP + hook logging)
- **#4 Rule arbitration** — Grok-first, port with gates
- **#5 Codex hard-fail** — platform-agnostic

3.5.0 is **loop closure on Grok**, not governance complete everywhere.

**On Grok:** governance README → governance OS. **Everywhere else:** honest orchestrator brain until 3.5.1 ports the teeth.

Still not auto-spawn (platform constraint). Defensible for multi-agent eng with audit needs **when host is named**.
