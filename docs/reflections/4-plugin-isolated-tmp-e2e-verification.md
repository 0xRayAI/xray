**Discover / Uncover / Learn / Hmm / Wow (testing phase focus)**:
- **Discover**: Subagents + monitor tool + per-cwd isolation is a *meta* demonstration of the project's own value claim (autonomous engine + observability + 3-subsystem). One command spawns 4 specialized verifiers that independently exercise the exact consumer path (pack → tgz install → npx plugin install → hooks/MCP/governance surface) while streaming the ground-truth activity.log. The hermes local log (217 lines of bridge + quality-gate + mcp nudges + post-processors) and openclaw one (hooks capturing real write_file marker, integration lifecycle) are richer than many unit tests.
- **Uncover**: Framework logging is cwd-relative (`join(cwd, "logs/framework/activity.log")`) + often async/fire-and-forget or gated; short validation e2es (grok/opencode) frequently produce 0 bytes in the "expected" tmp log even when events fire elsewhere (source monitor, ~/.grok, .opencode/logs/xray-plugin-*.log, plugin-tool-events companions). This is not a bug but explains why "monitoring the logs" required both the persistent source tail *and* subagent `tail` + `read` inside each cwd. Also uncovered that openclaw e2e Phase 13 explicitly asserts the activity.log + hook marker path — and it passed.
- **Learn**: "Triage. Fix. Retest. Until 100%" works beautifully when the flake is re-runnable (hermes codex_check stabilized); when the 1-fail is pure external infra (billing on the model the gateway is configured to use), the loop correctly identifies it as non-code and stops. Subagent reports are higher signal than raw e2e stdout because they combine exec + code inspection (e.g. confirmed exact `knowledge-skill-base.js` import of frameworkLogger, bridge.mjs size, installFor* source paths) in the isolated context.
- **Hmm**: The 2 grok skips (orchestrator/enforcer "not registered in Grok plugin .mcp.json") are intentional per the e2e + .mcp.json writer, yet Phase 7 still asserts reachability via CLI entrypoints. This is clean design but the test counts them as skips — future e2e polish could make the registration complete or the skip message even louder. Also, "using-fallback-path" warnings are noisy but harmless (consumer package.json vs. monorepo expectations).
- **Wow**: All 4 plugins' *core prod surfaces* (governance via Dynamo Solar in hook, full MCP (governance/skills/orchestrator/enforcer + 45 skills via skill-invocation), pre/post quality gates + 4 post-processors, activity logging + routing, bridge handoff to external CLIs) survived `npm pack` (1.4MB tgz with 1026 files) + isolated consumer npm install + the per-plugin `npx 0xray <p> install` + live external CLI exercise with **zero code defects found**. The subagent for openclaw even exercised the full integration lifecycle (init → hooks wiring to MCP clients for code-review/security-audit/etc. → shutdown) and confirmed the written config + activity.log assertions. This is the OS layer working as claimed under real published artifact conditions.
