# 0xRay Framework - Version Selection Guide

## 🚀 Framework Versions

Choose the right 0xRay Framework version for your team's needs and development workflow.

### **Framework Lite** (Recommended for Most Teams)

**80% Protection, 30% Complexity**

Perfect for AI-assisted development that needs essential safeguards without overwhelming complexity.

#### Key Features

- **Setup Time**: 5 minutes
- **Error Prevention**: 80% effective
- **Maintenance**: Minimal (minutes/month)
- **Team Size**: 1-20 developers
- **Integration**: Seamless with existing workflows

#### Protection Levels

- ✅ **Type Safety**: Comprehensive type checking
- ✅ **Bundle Monitoring**: Size limits (&lt;2MB)
- ✅ **Coverage Tracking**: Test coverage >85%
- ✅ **Duplication Control**: Code duplication &lt;5%
- ✅ **Security Scanning**: Basic vulnerability detection

#### When to Choose Lite

- Small to medium teams
- Fast-paced development
- Existing CI/CD pipelines
- Need essential AI safeguards
- Prefer simplicity over maximum protection

---

### **Framework Full** (Maximum Protection)

**90% Protection, Maximum Safeguards**

Complete implementation of all Universal Development Codex principles for maximum error prevention.

#### Key Features

- **Setup Time**: 30 minutes
- **Error Prevention**: 90% effective
- **Maintenance**: Moderate (hours/month)
- **Team Size**: 5-50+ developers
- **Integration**: Comprehensive workflow integration

#### Protection Levels

- ✅ **All Lite Features** (100% coverage)
- ✅ **Advanced Monitoring**: Real-time performance tracking
- ✅ **Code Rot Prevention**: Architectural violation detection
- ✅ **Multi-Agent Validation**: Cross-agent consensus checking
- ✅ **Progressive Enhancement**: Incremental capability building
- ✅ **State Management**: Global state consolidation
- ✅ **Single Source of Truth**: Centralized configuration management

#### When to Choose Full

- Large development teams
- Critical system development
- Complex architectural requirements
- Need maximum error prevention
- Willing to invest in comprehensive setup

## 🎯 Decision Framework

### Quick Assessment

Answer these questions to choose the right version:

1. **Team Size**: < 10 developers? → **Consider Lite**
2. **Project Criticality**: Life-critical systems? → **Choose Full**
3. **Setup Time**: Need immediate deployment? → **Choose Lite**
4. **Maintenance Budget**: Limited DevOps resources? → **Choose Lite**
5. **Architecture Complexity**: Simple CRUD apps? → **Choose Lite**
6. **AI Integration**: Heavy AI-assisted development? → **Choose Full**

### Recommended Combinations

- **Solo Developer**: Framework Lite
- **Small Startup (2-5 devs)**: Framework Lite
- **Growing Company (5-20 devs)**: Framework Lite → Full migration
- **Enterprise (20+ devs)**: Framework Full
- **Critical Systems**: Framework Full

---

## 🔄 Migration Path

### From Lite to Full

Framework Lite projects can migrate to Full when:

- Team size exceeds 20 developers
- Project complexity increases significantly
- Higher error prevention becomes critical

**Migration Process**:

1. Backup current configuration
2. Install additional Full components
3. Configure advanced monitoring
4. Train team on additional agents
5. Update CI/CD pipelines
